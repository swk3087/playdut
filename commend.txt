너는 시니어 풀스택 엔지니어다.

목표:
웹에서 동작하는 “엔트리(Entry) 프로젝트 고속 실행기”를 구현한다.
공식 EntryJS 플레이어를 사용하지 않고
프로젝트 JSON을 받아 IR → bytecode VM으로 실행한다.
목표는 엔트리보다 빠르게 동작하는 독립 실행기다.

중요 조건:
- 로컬에 참고용 디렉토리: "하위/entry-js"
- 이 디렉토리는 구조 참고용이며 결과물에 포함하지 않는다.
- 반드시 .gitignore에 추가한다.
- entry-js 코드를 복붙하지 말고 구조만 참고해 새 런타임을 구현한다.

===========================
하드웨어 블록 정책 (중요)
===========================

다음은 전부 구현하지 않는다:
- 아두이노/엔트리봇/로봇 블록
- 블루투스
- IoT
- 센서 하드웨어 입력
- 외부 장치 제어
- AI 하드웨어 카메라
- 마이크/음성 인식 하드웨어 블록

처리 방식:
- 하드웨어 블록은 컴파일 단계에서 감지
- 설정값에 따라:
  (1) 해당 스크립트 중단 + 경고 로그
  (2) noop 처리 + 경고
- 런타임이 절대 크래시하지 않도록 보호

===========================
아키텍처
===========================

Monorepo:
  /server (Node + Express + TS)
  /web (Vite + React + TS)

렌더:
  PixiJS (WebGL)

런타임:
  blocks → IR → bytecode VM
  블록 트리 런타임 해석 금지
  반드시 로딩 시 컴파일

스케줄러:
  Thread = { id, pc, stack, sleepUntilMs, waitingEvent, isDone }
  ready 큐 기반
  requestAnimationFrame 사용
  sleeping/waiting은 skip

충돌:
  spatial hash + AABB

오디오:
  WebAudio
